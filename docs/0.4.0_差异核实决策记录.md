## #1 / #2 / #3 — `load()` / `list()` / `changeSeed()` 同步 vs 异步

| 方法 | 设计文档 | TS | JS | Python | Go |
|------|---------|----|----|--------|-----|
| `load()` | Promise | 同步 | 异步 | 同步 | 同步 |
| `list()` | Promise | 同步 | 异步 | 同步 | 同步 |
| `changeSeed()` | Promise | 同步 | 异步 | 同步 | 同步 |

**决策**：设计文档改为同步。JS SDK 允许保持异步作为平台例外。

**原因**：
1. 三个方法都是纯本地操作（读文件、扫目录、重加密私钥），不涉及网络IO
2. 毫秒级完成，无需异步
3. 同步API更简洁，调用方不需要 await
4. TS/Python/Go 三个SDK已经是同步实现，JS 是唯一的异步版本
5. JS 保持异步的原因：底层存储是 IndexedDB（浏览器原生 API），所有操作均为异步，无同步版本，**无法改为同步，属于平台硬约束**

**行动项**：
- [ ] 更新设计文档 `load()` / `list()` / `changeSeed()` 签名为同步
- [ ] 设计文档中注明 JS SDK 因浏览器环境限制保持异步

## #4 — `resolve()` 等方法返回类型精度

| 方法 | 设计文档 | TS/JS | Python | Go |
|------|---------|-------|--------|-----|
| `resolve()` | 强类型 | `Record<string,unknown>` | `dict[str,Any]` | ✅ 强类型 |
| `fetchAgentMd()` | 强类型 | `Record<string,unknown>` | `dict[str,Any]` | ✅ 强类型 |
| `headAgentMd()` | 强类型 | `Record<string,unknown>` | `dict[str,Any]` | ✅ 强类型 |
| `checkAgentMd()` | 强类型 | `Record<string,unknown>` | `dict[str,Any]` | ✅ 强类型 |
| `diagnose()` | 强类型 | `Record<string,unknown>` | `dict[str,Any]` | ✅ 强类型 |
| `renewCert()` | 强类型 | `Record<string,unknown>` | `dict[str,Any]` | ✅ 强类型 |
| `rekey()` | 强类型 | `Record<string,unknown>` | `dict[str,Any]` | ✅ 强类型 |

**决策**：所有 SDK 均改为强类型返回。Go 已完成，TS/JS 补充 interface，Python 补充 TypedDict。

**原因**：运行时字段已确定且稳定，弱类型对调用方不友好，IDE 无法提示，容易出错。

**强类型定义（以 TS 为基准，各 SDK 对应实现）**：

```typescript
type ResolveResult = {
  aid: AID;
  agent_md?: { content: string; verification: { status: string; reason?: string }; cert_pem: string };
  source: { cert_from_cache: boolean; agent_md_fetched: boolean };
};
type FetchAgentMdResult = {
  aid: string; content: string;
  verification: { status: string; reason?: string };
  cert_pem: string; etag: string; last_modified: string;
};
type HeadAgentMdResult = { aid: string; found: boolean; etag: string; last_modified: string; content_length: number };
type CheckAgentMdResult = { aid: string; local_found: boolean; remote_found: boolean; local_etag: string; remote_etag: string; needs_update: boolean; ttl_days: number };
type DiagnoseResult = { aid: string; status: string; local_valid: boolean; remote_registered: boolean; suggestions: string[]; local: Record<string, unknown>; remote: Record<string, unknown> };
type RenewCertResult = { renewed: true; new_cert_not_after: Date; new_fingerprint: string };
type RekeyResult = { rekeyed: true; new_cert_not_after: Date; new_fingerprint: string };
```

**行动项**：
- [ ] TS/JS `aid-store.ts` 定义上述 interface，替换 `Record<string, unknown>`
- [ ] Python `aid_store.py` 定义对应 TypedDict，替换 `dict[str, Any]`
- [ ] 更新设计文档类型定义章节

## #5 — AIDStore 额外构造参数

| 参数 | 设计文档 | 各SDK现状 | 决策 |
|------|---------|----------|------|
| `aunPath` | ✅ 有 | 全部有 | 保留 |
| `encryptionSeed` | ✅ 有 | 全部有 | 保留 |
| `deviceId?` | ✅ 有 | 全部有 | 保留 |
| `slotId?` | ✅ 有 | 全部有 | 保留 |
| `verifySsl?` | ❌ 无 | 全部有 | **保留，补充进设计文档** |
| `rootCaPath?` | ❌ 无 | 全部有 | **保留，补充进设计文档** |
| `debug?` | ❌ 无 | 全部有 | **保留，补充进设计文档** |
| `discoveryPort?` | ❌ 无 | 全部有 | **移除** |

**决策**：
- 保留 `verifySsl`、`rootCaPath`、`debug`，补充进设计文档
- 移除 `discoveryPort`：服务发现不应指定端口，端口应由服务发现机制自动解析

**原因**：
- `verifySsl`：测试环境需要关闭 SSL 校验，实用参数
- `rootCaPath`：私有部署需要自定义根证书，必要参数
- `debug`：控制日志级别，调试必备
- `discoveryPort`：硬编码端口违背服务发现的设计意图，应由 gateway URL 或 DNS SRV 等机制自动确定

**行动项**：
- [ ] 设计文档 AIDStore 构造参数补充 `verifySsl?`、`rootCaPath?`、`debug?`
- [ ] 各 SDK 移除 `discoveryPort` 参数及相关逻辑（涉及 agent.md URL 拼接、gateway 发现）
- [ ] 移除后 agent.md URL 拼接改为从 gateway URL 自动推导端口

## #9 — AUNClient 构造签名

| | 设计文档 | TS | JS | Python | Go |
|---|---------|----|----|--------|-----|
| 构造签名 | `constructor(aid?: AID)` | `(options?)` 或 `(aid, options?)` | 同TS | `(aid=None, *, options=None)` | `(first any, options...)` |

**决策**：收紧为设计文档的纯粹设计——`constructor(aid?: AID)`，移除 options 重载。

**机制**：
- AIDStore 在 `load()` 时将必要配置（`aunPath`、`encryptionSeed`、`verifySsl`、`rootCaPath`、`debug`）作为只读属性附加到 AID 对象
- AUNClient 从 AID 对象上读取所有配置，不再单独接受 options 参数
- AUNClient 不能脱离 AIDStore 独立使用

**原因**：
1. 职责清晰：AIDStore 是配置管理器，AUNClient 只管连接
2. 避免配置散落：不会出现 AIDStore 和 AUNClient 配置不一致的问题
3. API 简洁：构造函数只有一个参数，语义明确

**行动项**：
- [ ] 各 SDK 移除 AUNClient 的 options 重载，只保留 `constructor(aid?: AID)`
- [ ] AID 对象补充只读属性：`aunPath`、`encryptionSeed`、`verifySsl`、`rootCaPath`、`debug`（由 AIDStore.load() 注入）
- [ ] 设计文档 AID 只读属性表补充上述字段
- [ ] 设计文档明确 AUNClient 必须通过 AIDStore 创建的 AID 使用

## #12 — `authenticate()` 状态约束

| | 设计文档 | TS/JS | Python | Go |
|---|---------|-------|--------|-----|
| 允许状态 | 仅 Standby | Standby + Authenticated | 仅 Standby | 无显式检查 |

**决策**：所有 SDK 统一为仅 Standby 状态可调，对齐设计文档。

**原因**：
- token refresh 失败时，SDK 内部走断线重连路径（→ RetryBackoff → Reconnecting），重连时自动重新 authenticate，不需要应用层手动调用
- 允许在 Authenticated 状态重新 authenticate 是多余的宽松，增加状态机复杂度
- 应用层无需关心 token 刷新细节，这是 SDK 内部职责

**行动项**：
- [ ] TS/JS 收紧 `authenticate()` 状态检查：移除 Authenticated 状态的允许
- [ ] Go 补充显式状态检查：仅 Standby 可调

## #13 — `connect()` 参数

| | 设计文档 | TS/JS | Python | Go |
|---|---------|-------|--------|-----|
| 参数 | `{ gateway? }` | `connect(options?: RpcParams)` | `connect(options?: dict)` | `Connect(ctx, opts...)` |

**决策**：`connect()` 无参数，签名改为 `connect(): Promise<void>`。

**机制**：
- `authenticate()` 执行后，将结果缓存到 AUNClient 私有字段：`_gatewayUrl`、`_accessToken`、`_refreshToken`
- `connect()` 直接使用缓存的 gateway URL 和 token 建立连接
- 若未 authenticate，`connect()` 内部自动先调用 `authenticate()`（使用 AID 中的配置发现 gateway）
- cached token/URL 的复用也在 authenticate 流程内部完成

**原因**：gateway URL 是认证结果的一部分，不应由调用方传入；connect 只管建连接，不管配置。

**行动项**：
- [ ] 各 SDK `connect()` 移除参数
- [ ] 设计文档更新 `connect()` 签名为无参数，并说明 gateway URL 来源
